# 스마트모빌리티공학실험2 기말 프로젝트 보고서

**제출 알고리즘:** v30 (Isotonic+MLP 거리 보정 → 비대칭 가중 Huber 삼변측량 → 2-pass 위치 아핀)  
**자체 평가:** 700 UE, 5-Fold CV (seed=42), 채점 hidden 300 UE는 미사용

---

## 1. 모티베이션 & 인트로

### 1.1 중간 프로젝트에서 얻은 교훈

중간 과제에서는 약 12 m×18 m 타일 위에서 6쌍 UWB·Wi-Fi 거리를 융합해 2D 위치를 추정했다. 재실행 기준 UWB 단독 RMSE는 약 2.8 m 수준이었으나, 분산 필터·MAD 제거 후에도 앵커별 체계적 과소 측정(약 −0.8~−3.6 m)이 남았고, Wi-Fi는 바닥 성능(약 1.55 m)에 가깝게 작동했다. 결정적 교훈은 세 가지였다. 첫째, **강건 손실(Huber)과 기하학적 삼변/최소제곱 구조**를 버리지 말 것. 둘째, **검증셋에 맞춘 τ·K 튜닝(V9·V10)** 은 기말 hidden 평가에서 불리할 수 있으므로, fold 단위 학습 분리가 필요하다. 셋째, **UWB 이상치를 행 삭제로 “정제”하는 전략**은 기말 합성 RTT 환경으로는 이관하지 않는다.

### 1.2 기말 데이터에서 본 문제 구조

기말 `InF_DH_FR1.mat`(채점 시 `DH_FR1.mat`)은 18기지국 RTT 한 종류만 주어지며, UE 700명의 위치 span은 설계상 약 120 m×60 m(±60 m×±30 m)이다. EDA에서 관측 거리 d̂와 기하학적 진거리 d_true의 Pearson 상관은 약 0.815로 단조성은 유지되나, 잔차 평균은 약 +15.9 m, 양의 편향 비율은 약 81%로 **NLOS에 따른 과대 측정 RTT** 가 dominant했다. 전역 아핀 d′=αd+β만으로는 α≈0.61, β≈9.2 수준의 뭉개진 보정에 그쳐, BS별 단조(Isotonic)와 잔차 학습이 필요함을 확인했다.

### 1.3 제안 파이프라인 (high-level)

본인의 최종 방법(v30)은 **“거리 도메인에서 물리적으로 해석 가능한 보정을 먼저 하고, 기하 삼변으로 위치를 복원한 뒤, 위치 도메인에서 소규모 아핀으로 잔차를 흡수”** 하는 3단 구조이다. 중간 과제의 Wi-Fi 바닥(1.55 m)과 직접 비교하는 것은 관측 스케일·합성 방식이 달라 **unfair** 하므로, 기말에서는 **동일 RTT·동일 5-Fold CV** 위에서 Huber 삼변(B1)부터 점진적으로 모듈을 더하는 **공정한 ablation** 만을 보고한다.

## 2. 알고리즘 설명

본 절의 기호: UE 위치 p=(x,y), 기지국 k의 좌표 b_k, 관측 RTT 거리 벡터 d̂∈R^18, 보정 후 거리 d∈R^18.

### 2.1 1단계 — BS별 Isotonic 거리 보정

각 기지국 k에 대해 학습 fold의 (d̂_k, d_true,k) 쌍으로 단조 회귀(Isotonic Regression)을 적합한다. 추론 시 d̂_k를 학습된 단조 곡선 f_k에 통과시켜 d_iso,k=f_k(d̂_k)를 얻는다. 이는 **음의 기울기 보정을 금지**하면서 NLOS 편향의 BS별 비선형 왜곡을 흡수한다. 중간 프로젝트의 고정 앵커 bias 테이블과 달리, **데이터 기반 단조 매핑**이라는 점이 본인 설계의 핵심이다.

### 2.2 2단계 — Isotonic 위의 MLP 잔차 보정

18차원 벡터 d_iso를 입력으로 하는 얕은 MLP(은닉 64, dropout 0.25)를 학습한다. 출력은 **잔차 형태**로 d_out = d_iso + Δ(d_iso)이며, MSE로 d_true에 맞춘다. 입력은 채널별 z-score 정규화 후 학습한다. 이 2단 구조(Isotonic+MLP)는 v13(MLP 단독, CV RMSE 약 10.81 m)보다 안정적이며, v16에서 CV RMSE 약 9.28 m까지 개선되었다. **물리 해석 가능한 1차 보정(Isotonic) 위에만 신경망을 얹는다**는 점이 타 학생의 “좌표 직접 회귀”와 구별되는 설계이다.

### 2.3 3단계 — soft 가중·비대칭 Huber 삼변측량

보정 거리 d로 UE 위치 p를 구한다. 앵커 k에 대한 잔차는 r_k(p)=||p−b_k||−d_k. 목적함수는 Huber 손실의 합이며, **예측 거리가 관측보다 길 때(r_k>0)** 에만 추가 가중 w_pos=5를 곱한 **비대칭 가중 제곱근 잔차**를 사용한다. 이는 d̂가 체계적으로 짧게 측정되는 NLOS(양의 bias) 상황에서, 삼변 해가 관측보다 “멀리” 가는 방향을 억제한다. v14에서 동일 아이디어를 Isotonic 단독 스택에 적용했을 때는 약 10.33 m로 악화되었으나, **v25 스택(MLP+아핀) 이후에 결합**하자 v30에서 약 7.996 m까지 내려갔다. 이 **조건부 모듈 결합**이 본인 알고리즘의 가장 참신한 부분이다.

거리 가중은 γ=1.0일 때 w_k ∝ 1/d_k (정규화)로 가까운 BS를 상대적으로 강조한다. 최적화는 scipy least_squares, Huber f_scale=1.0, 위치 box는 [-62,62]×[-32,32] m로 클리핑한다.

### 2.4 4단계 — 2-pass 위치 아핀 (train-only fitting)

삼변 결과 p^(1)에 대해, 학습 UE만으로 p_true ≈ M·[x_hat, y_hat, 1]^T (2×3 아핀 M)을 최소제곱 적합한다. 추론 시 p = M·[x_hat,y_hat,1]^T 후 box 클리핑. **1pass 삼변으로 coarse 위치를 만든 뒤, 2pass에서 선형 잔차를 흡수**하므로, v18처럼 d̂→(x,y) 직접 회귀(CV 약 38 m)와 달리 **기하 제약을 유지**한다. v25에서 CV RMSE 약 8.76 m, v30에서 약 7.996 m.

### 2.5 학습·추론 분리 (채점 무결성)

train.py는 700 UE 전체 또는 각 CV fold의 train 560명으로만 Isotonic·MLP·아핀을 학습한다. main.py는 DH_FR1.mat의 d̂, p_bs만 사용하며 **정답 p는 읽지 않는다**. 학습된 Isotonic knot, MLP 가중치, 아핀 M, pipeline 하이퍼(γ, w_pos)는 model_mlp.pt 번들에 저장된다.

### 2.6 참고 문헌과의 차이 (Reference 선행)

3GPP TR 38.901 부록에서 Indoor Hotspot·InF-DH 채널은 클러스터 지연·NLOS 확률을 모델링한다. 본인 방법은 **채널 시뮬레이터를 구현하지 않고**, 제공된 d̂에서 **경험적 단조·잔차·비대칭 기하 최적화**로 편향을 제거한다. 논문/표준의 “NLOS state”를 명시적으로 추정하지 않으며, 대신 **r_k>0 방향 비대칭 penalty**로 동일 현상을 약하게 반영한다.

## 3. Agent AI 활용 방안

### 3.1 사용 도구

- **Cursor (Claude 기반 Agent):** 코드 탐색, 실험 스크립트 초안, 문서 초안
- **Gemini:** 3GPP InF-DH·NLOS 편향 관련 **아이디어 브레인스토밍** (과장 주장은 본인이 실험으로 걸러냄)

### 3.2 본인이 직접 수행·결정한 영역 (핵심)

| 영역 | 본인 역할 |
|------|-----------|
| 문제 정식화 | 중간 고찰→기말 RTT-only 파이프라인 설계, hidden 300명 고려한 CV·hold-out 정책 |
| EDA 해석 | +15.9 m bias, 81% 양의 잔차, BS·공간별 꼬리 UE 분석 후 **Isotonic→MLP→삼변→아핀** 순서 확정 |
| 모듈 채택·폐기 | HardGate 폐기, PosMLP(v18)·quantile(v27)·top_k(v31) 등 **직접 ablation 후 제거** |
| v30 핵심 가설 | “비대칭 Huber는 Isotonic 단독이 아니라 **v25 스택 뒤**에만 유효”를 combination review로 확인 |
| 하이퍼 선택 | asym w∈{1,1.5,2,3,5,8} 그리드에서 **w=5** 선택 (w=8은 7.999 m로 과적합 징후) |
| 무결성 | fold별 calib 분리, main.py GT 미사용, top_k 보정 후 적용 버그 수정 |
| 보고서·제출 | 과제 규격(단일 repo, model.pt 번들, train/main 일치)에 맞게 **구조 재설계** |

### 3.3 Agent AI가 보조한 영역

- sklearn·torch API가 헷갈릴 때의 **함수 시그니처 확인**
- 긴 실험 로그를 표로 정리하는 **초안 문장** (수치는 본인이 JSON·CSV로 재검증)
- 반복적인 버전 등록 보일러플레이트

### 3.4 역할 구분 원칙

Agent가 제안한 아이디어 중 **CV에서 악화되거나 누수가 있는 것은 채택하지 않았다.** 최종 v30과 제출 `main.py`/`train.py`의 **수식·실행 순서·하이퍼**는 본인이 실험 표를 근거로 확정했으며, Agent는 “구현 속도”에 가깝게 사용했다.

### 4.7 자체 평가의 공정성·한계

**공정성:** (1) 모든 버전 동일 5-Fold, seed=42. (2) fold마다 calib·MLP·아핀 재학습, validation UE는 학습 통계에 미포함. (3) main.py는 GT 미사용. (4) hold-out 560/140에서 v16·v17은 약 9.31 m로, CV 8.9 m 대비 +0.52 m 수준의 gap만 있어 **심각한 누수는 없다**고 판단.

**한계:** (1) asym w=5는 OOF 그리드로 고른 값이라 hidden에서 약간 낙관적일 수 있다. (2) far UE(GT 반경 큰 구간) RMSE는 여전히 10 m 이상으로, **전역 8 m대가 현실적 상한**에 가깝다. (3) 채점 hidden 300명 성능은 제출 시점에 알 수 없다.

### 4.8 장점·단점·향후

**장점:** 해석 가능한 단조 보정 + 기하 삼변 + 소규모 위치 아핀; **비대칭 Huber를 마지막에만 얹는 조합 설계**; train/main·보고서·model.pt 번들 일치.

**단점:** MLP·아핀·w_pos 등 하이퍼가 많아 구현 복잡도가 높다; far tail UE는 구조적으로 어렵다.

**Future work:** min(d̂) 기반 zone별 Isotonic(v21)과 아핀의 재조합, fold-nested asym 튜닝, 거리 보정 목표와 위치 RMSE의 multi-task 정렬.
